# types-reader

A helper library for writing Rust procedural macros.

Proc-macro authors spend most of their time on the same boring job: taking the token soup inside
`#[my_attr(...)]` and turning it into typed Rust values, with decent compiler errors when the user
gets it wrong. `types-reader` does that job. It parses the tokens into a small JSON-like object
model and then lets you declare the shape you expect as an ordinary Rust struct:

```rust
use types_reader::macros::*;

#[attribute_name("http_query")]
#[derive(MacrosParameters)]
pub struct HttpQueryAttribute<'s> {
    pub name: Option<&'s str>,
    pub description: &'s str,
    #[has_attribute]
    pub trim: bool,
}
```

That struct now parses `#[http_query(name: "id", description: "Record id", trim)]`, reports unknown
parameters and type mismatches as proper `syn::Error`s with spans, and never makes you touch a
`TokenTree`.

## The three crates

| Crate | What it is |
|---|---|
| `types-reader-core` | All the types: `TokensObject`, `Attributes`, `StructProperty`, `PropertyType`, `TypeName`, the value wrappers. |
| `types-reader-macros` | The three macros: `MacrosParameters`, `MacrosEnum`, `attribute_name`. |
| `types-reader` | The facade you should depend on. It is two lines: `pub use types_reader_core::*;` and `pub extern crate types_reader_macros as macros;`. |

**Depend on the `types-reader` facade.** The code generated by the macros hardcodes the path
`types_reader::` (e.g. it emits `impl<'s> TryFrom<&'s types_reader::TokensObject> for ...`), so the
name `types_reader` must resolve in your crate. Depending on the facade makes that automatic.

If you depend on `types-reader-core` directly, you must add the alias yourself, or nothing compiles:

```rust
use types_reader_core as types_reader; // required when using core without the facade
```

## Installation

```toml
[dependencies]
types-reader = { tag = "0.5.1", git = "https://github.com/MyJetTools/types-reader.git" }
```

Then, in a proc-macro crate:

```rust
use types_reader::macros::*;   // MacrosParameters, MacrosEnum, attribute_name
use types_reader::TokensObject; // the object model
```

## How you actually use it

There are exactly two shapes, and every downstream macro in the wild is one of them.

### 1. Attribute macro — `#[my_attr(...)] struct Foo`

You get the attribute's tokens; parse them into your parameter struct:

```rust
#[proc_macro_attribute]
pub fn http_route(attr: TokenStream, input: TokenStream) -> TokenStream {
    match build(attr, input) {
        Ok(result) => result,
        Err(err) => err.to_compile_error().into(),   // spans survive, user sees a real error
    }
}

fn build(attr: TokenStream, input: TokenStream) -> Result<TokenStream, syn::Error> {
    let ast: syn::DeriveInput = syn::parse(input)?;

    let attr: proc_macro2::TokenStream = attr.into();
    let tokens_object: TokensObject = attr.try_into()?;              // tokens -> object model
    let params: ActionParameters = (&tokens_object).try_into()?;     // object model -> your struct

    let route = params.route;
    Ok(quote::quote!( /* ... */ ).into())
}
```

### 2. Derive macro — `#[derive(MyDerive)] struct Foo { #[my_attr(...)] field: T }`

You walk the fields and read each field's attributes:

```rust
fn generate(ast: &syn::DeriveInput) -> Result<TokenStream, syn::Error> {
    let fields = types_reader::StructProperty::read(ast)?;   // Vec<StructProperty>

    for prop in &fields {
        if prop.attrs.has_attr("ignore") {
            continue;
        }

        // The target type drives the lookup - no turbofish, no string literal.
        // Uses MacrosAttribute::NAME, set by #[attribute_name("http_query")].
        let attr: Option<HttpQueryAttribute> = prop.try_get_attribute()?;
        if let Some(attr) = attr {
            // attr.name, attr.description, attr.trim ...
        }
    }

    Ok(quote::quote!( /* ... */ ).into())
}
```

## The parameter grammar

This is what may appear inside `#[my_attr( ... )]`. It is *not* Rust syntax and it is *not* JSON —
it is its own small grammar:

```text
#[my_attr]                                  no content
#[my_attr("just-a-value")]                  a single, unnamed value
#[my_attr = "just-a-value"]                 same thing, `= value` form (this is what /// expands to)
#[my_attr(name: "value")]                   named parameter, ':' separator
#[my_attr(name = "value")]                  named parameter, '=' separator (identical meaning)
#[my_attr(flag)]                            named parameter with no value (a flag)
#[my_attr(a: 1, b: 2)]                      ',' separates parameters
#[my_attr(a: 1; b: 2)]                      ';' separates parameters too (identical meaning)
#[my_attr(nested(a: 1, b: 2))]              list form - a nested object
#[my_attr(nested: {a: 1, b: 2})]            braces - also a nested object
#[my_attr(items: [1, 2, 3])]                brackets - an array
#[my_attr(items: [{a: 1}, {a: 2}])]         array of objects
```

Values may be strings (`"x"`), integers (`5`, `-1`), floats (`1.5`, `-1.5`), and booleans
(`true` / `false`).

**Mixing a single value with named parameters is an error.** `#[my_attr("v", other: true)]` is
rejected: once you name one parameter you must name them all.

The result is a `TokensObject`, which is one of three things:

| Variant | Produced by | Test with |
|---|---|---|
| `Value` | `#[my_attr("v")]`, `#[my_attr]` | `unwrap_as_value()` |
| `Object` | `#[my_attr(a: 1, b: 2)]` | `is_object()` |
| `Vec` | `[1, 2, 3]` | `is_vec()` / `unwrap_as_vec()` |

## `#[derive(MacrosParameters)]`

Applied to a **struct with named fields**. It generates:

```rust
impl<'s> TryFrom<&'s types_reader::TokensObject> for YourStruct<'s> {
    type Error = syn::Error;
    fn try_from(value: &'s types_reader::TokensObject) -> Result<Self, Self::Error> { ... }
}

impl YourStruct<'_> {
    // rejects any parameter whose name is not a field - this is your typo protection
    pub fn check_fields(tokens_object: &types_reader::TokensObject) -> Result<(), syn::Error> { ... }
}
```

> **The lifetime must be named `'s`.** The generated `try_from` signature hardcodes `&'s`. A struct
> declared `struct Foo<'a>` fails to compile with `error[E0261]: use of undeclared lifetime name 's`.
> A struct with no lifetime at all is fine (the `'s` is synthesized). Type and const generics are not
> supported.

### Field types

| Field type | Accepts | Notes |
|---|---|---|
| `&'s str`, `String` | `name: "value"` | Requires a quoted string unless `#[allow_ident]` |
| `bool` | `name: true` | An explicit literal. For presence-only, use `#[has_attribute]` |
| `u8`…`u64`, `i8`…`i64`, `usize`, `isize` | `name: 42` | Not range-checked, see gotchas |
| `f32`, `f64` | `name: 1.5` | Requires a decimal point |
| `Option<T>` | parameter may be absent | Absent or valueless yields `None` |
| `Vec<T>` | `name: [a, b]` | Required; use `Option<Vec<T>>` to make it optional |
| A nested `MacrosParameters` struct | `name(a: 1)` or `name: {a: 1}` | Composes recursively |
| A `MacrosEnum` enum | `name: "Case"` | See below |
| `&'s TokensObject` | anything | The raw subtree — parse it yourself |
| `&'s ObjectValue`, `&'s OptionalObjectValue` | anything | The raw value |
| `AnyValue<'s>` | any literal, or bare `name` | When you don't care about the type |
| `MaybeEmptyValue<T>` | value or nothing | Distinguishes "absent" from "present but empty" |

### Field helper attributes

| Attribute | Meaning |
|---|---|
| `#[has_attribute]` | **`bool` fields only.** The field becomes "was this parameter named at all?". Applying it to a non-bool field is a compile error. |
| `#[default]` | This field may be supplied as the attribute's *single unnamed value*: `#[my_attr("x")]` fills it. It does **not** mean "has a default value". |
| `#[allow_ident]` | Accept a bare word (`method: GET`) instead of requiring `"GET"`. Also coerces numbers/bools to their source text. |
| `#[any_value_as_string]` | Currently an **exact synonym** of `#[allow_ident]` — the two are OR'd into one flag in the codegen. |

### A complete example

```rust
use types_reader_core as types_reader;
use types_reader_macros::MacrosParameters;

#[derive(MacrosParameters)]
pub struct SubParameters<'s> {
    pub sub_parameter1: &'s str,
    pub sub_parameter2: &'s str,
}

#[derive(MacrosParameters)]
pub struct HttpActionResult<'s> {
    pub status_code: u16,
    pub description: SubParameters<'s>,          // nested struct
    #[allow_ident]
    pub model: Option<&'s str>,
    pub as_vec: Vec<SubParameters<'s>>,
    pub as_vec_opt: Option<Vec<SubParameters<'s>>>,
}
```

## `#[derive(MacrosEnum)]`

Applied to an enum whose cases are **unit cases** or **single-element tuple cases**. Named-field
cases (`Case { a: u8 }`) are rejected with a proper error.

```rust
#[derive(MacrosEnum, Debug)]
pub enum ShouldBeAuthorized {
    Yes,
    No,
    YesWithClaims(Vec<String>),
}
```

- A **unit case** matches a quoted string equal to the case name: `authorized: "Yes"`.
  Use `#[value("custom-text")]` on the case to match different text.
- A **tuple case holding a `Vec`** matches an array: `authorized: ["claim1", "claim2"]`.
- A **tuple case holding anything else** matches an object: `authorized: {a: 1}`.
- `#[default]` on a case generates an `impl Default`.

Generated API:

```rust
impl ShouldBeAuthorized {
    pub fn try_from_str(value: &str) -> Option<Self>;
    pub fn as_str(&self) -> &str;   // NOT generated if any case holds a Vec
}
// plus TryInto<ShouldBeAuthorized> for &ObjectValue, &OptionalObjectValue,
// &TokensObject and &dyn AnyValueAsStr
```

`as_str` is omitted when a `Vec` case exists (it would have no text to return for it), so the enum
above — which has `YesWithClaims(Vec<String>)` — gets `try_from_str` but no `as_str`.

An unknown string produces a `syn::Error` listing the supported values, not a panic:

```text
Unsupported value: Yess. Supported values are: 'Yes','No'
```

## `#[attribute_name]`

Stamps the attribute's own name onto the type:

```rust
#[attribute_name("http_query")]
#[derive(MacrosParameters, Clone)]
pub struct HttpQueryAttribute<'s> { /* ... */ }
```

generates

```rust
impl<'s> types_reader::MacrosAttribute for HttpQueryAttribute<'s> {
    const NAME: &'static str = "http_query";
}
```

This is what makes `prop.try_get_attribute::<HttpQueryAttribute>()?` work without repeating the
string, and what `attrs.has_attr(IgnoreAttribute::NAME)` reads.

Put it **above** `#[derive(...)]` — it re-emits the type and appends the impl. It currently requires
a named-field struct.

## Reading attributes off a struct you are deriving

`StructProperty::read(&ast)` gives one entry per named field, with the field's attributes already
parsed:

```rust
pub struct StructProperty<'s> {
    pub name: String,
    pub ty: PropertyType<'s>,
    pub field: &'s syn::Field,
    pub attrs: Attributes<'s>,
}
```

`Attributes` is a `HashMap<String, Vec<TokensObject>>` keyed by attribute name, with the usual
accessors:

```rust
attrs.has_attr("ignore")                        // -> bool
attrs.try_get_attr("my_attr")                   // -> Option<&TokensObject>
attrs.get_attr("my_attr")                       // -> Result<&TokensObject, syn::Error>
attrs.get_attrs("doc")                          // -> Result<&Vec<TokensObject>, syn::Error>, all occurrences
attrs.get_named_param("my_attr", "name")        // -> Result<&TokensObject, syn::Error>
```

You can also build it for the container itself:

```rust
let attrs = Attributes::new(&ast, &ast.attrs)?;
```

Doc comments land here too: `/// Hello` is expanded by the compiler into `#[doc = " Hello"]`, so it
appears as an attribute named `doc` whose value is the string. Use `get_attrs("doc")` to collect all
lines — `get_attr` returns only the first.

`PropertyType` classifies the field's Rust type (`U8`, `String`, `Bool`, `OptionOf(..)`, `VecOf(..)`,
`Struct(..)`, `HashMap(..)`, `RefTo { .. }`, …) so your codegen can branch on it.

## The object model by hand

When the derive is not enough, walk `TokensObject` directly:

```rust
let tokens: TokensObject = attr_token_stream.try_into()?;

let id: i32 = tokens.get_named_param("id")?.try_into()?;
let name: &str = tokens.get_named_param("name")?.try_into()?;

if let Some(opt) = tokens.try_get_named_param("optional") { /* ... */ }

for item in tokens.get_named_param("list")?.unwrap_as_vec()? { /* ... */ }
```

The naming convention across the crate:

| Prefix | Contract |
|---|---|
| `get_*` | Returns `Result<_, syn::Error>` — missing is an error |
| `try_get_*` / `try_*` | Returns `Option` or `Result<Option<_>>` — missing is `None` |
| `unwrap_*` | Asserts the shape; returns `Result<_, syn::Error>` (a few genuinely panic) |

`get_value_from_single_or_named("x")` is the one worth knowing: it accepts *either* `#[attr("v")]`
*or* `#[attr(x: "v")]`, which is what `#[default]` is built on.

Every error carries a span, so `err.to_compile_error()` underlines the offending token in the user's
editor rather than the whole macro.

## Gotchas and limitations

Verified against the current code. The first group is **surprising but by design**; the second group
are **known bugs** — if you hit one, fix the library rather than working around it.

### By design

- **`#[has_attribute]` tests presence, not value.** `#[my_attr(flag: false)]` sets the field to
  `true`, because only the name is examined. If you want a real boolean, use a plain `bool` field.
- **`#[default]` means "single unnamed value", not "default value".** Easy to misread.
- **`#[allow_ident]` and `#[any_value_as_string]` are the same thing today.** The codegen ORs them.
- **`;` and `,` are interchangeable**, as are `:` and `=`. There is no "one true" spelling.
- **`check_fields` only guards names you declared.** It is your typo protection and it is generated
  automatically — but only for `MacrosParameters` structs.

### Known bugs

- **A duplicated parameter silently wins.** `#[my_attr(id: 1, id: 2)]` parses to `id == 2` with no
  error — the parameters live in a `HashMap` with no occupancy check.
- **Numbers are not range-checked.** `id: 300` into a `u8` field silently yields `44`; `id: -1` into
  a `u64` yields `18446744073709551615`. Every accessor is an unchecked `as` cast over an `i64`.
- **String escapes are not decoded.** `msg: "a\nb"` yields the four characters `a\nb`, not a newline.
  Raw strings (`r#"..."#`) are rejected. Anything carrying a path, regex or format string is affected.
- **Path-style attributes break the derive.** A single `#[rustfmt::skip]` on any field of a derived
  struct fails the whole derive: only single-ident attribute names are understood.
- **Several user mistakes panic instead of erroring.** Deriving `MacrosParameters` on an enum or a
  tuple struct, `MacrosEnum` on a struct, or using an unsupported field type (`Tuple`, `Slice`,
  `Array`, `HashMap<&str, _>`, …) aborts the compiler with `proc-macro derive panicked` instead of a
  spanned diagnostic.
- **The lifetime must be `'s`** (see above).

## Repository layout

```
types-reader/            facade crate - depend on this
types-reader-core/       the types
  src/attributes.rs        #[attr] -> Attributes
  src/struct_property.rs   syn::DeriveInput -> fields + attrs
  src/structure_schema.rs  fields + parsed type name
  src/enum_case/           enum variants + attrs
  src/property_type/       classify a field's Rust type
  src/type_name.rs         struct name + generics/lifetimes, for impl blocks
  src/tokens_object/       THE object model: TokensObject, ObjectValue, value wrappers
  src/tokens_iterator/     token cursor underneath it all (internal in practice)
types-reader-macros/     MacrosParameters, MacrosEnum, attribute_name
tests/                   the de-facto specification - read these for more examples
```

The `tests/` crate is the best source of worked examples; `tests/src/test_from_real_life/` mirrors
patterns taken from real downstream macros.
